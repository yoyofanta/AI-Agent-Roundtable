# AI Agent 多角色圆桌对谈与剧本生成系统测试总结

## 一、项目概述

本项目为 **AI Agent 多角色圆桌对谈与剧本生成系统**，基于 Vue3、TypeScript、Spring Boot、H2 数据库和 DeepSeek API 实现，主要面向漫剧 / 短剧内容生产场景。

系统提供两种内容生成路径：

1. **直接生成方案**：用户输入题材、平台、受众、风格和具体需求，系统结合 Tool Calling 内容分析与 RAG 知识库生成剧本方案。
2. **多 Agent 协助生成**：用户选择 2-4 个 Agent 进行圆桌讨论，系统生成圆桌总结后，可一键带入剧本生成页面，最终生成完整剧本方案。

本次测试围绕系统核心功能、接口稳定性、AI 应用效果和项目演示完整性展开，目标是保证系统能够支撑面试展示和 GitHub 项目说明。

---

### 2. 直接生成方案流程

测试流程：


登录
→ 进入内容页
→ 点击直接生成方案
→ 输入题材、平台、受众、风格和具体需求
→ 点击先分析内容
→ Tool Calling 返回内容分析结果
→ 点击生成剧本方案
→ 页面展示最终剧本方案


测试结果：通过。

系统能够根据用户输入生成内容分析结果，包括冲突强度、情绪基调、推荐标签和优化建议。随后系统可以基于分析结果生成剧本方案，内容包含故事简介、人物设定、核心冲突、分集大纲、关键对白、标题标签和运营建议。



### 3. 多 Agent 协助生成流程

测试流程：


登录
→ 进入内容页
→ 点击多 Agent 协助生成
→ 选择 2-4 个 Agent
→ 输入圆桌议题
→ 开始圆桌讨论
→ 每个 Agent 依次发言
→ 生成圆桌总结
→ 点击带入剧本生成
→ 回到内容生成页
→ 基于圆桌总结生成最终剧本方案


测试结果：通过。

系统能够完成多 Agent 圆桌讨论流程。不同 Agent 可以基于自身角色设定进行发言，圆桌总结能够提炼主要观点。带入剧本生成后，最终结果会包含圆桌讨论结论、不同 Agent 观点采纳情况和最终剧本方案，与直接生成方案有明显区别。

---

### 4. 自定义 Agent 流程

测试内容包括：

* 新建自定义 Agent
* 设置角色名称
* 选择参考人格
* 设置性别
* 填写角色 Prompt
* 保存后自动加入圆桌

测试结果：通过。

系统支持创建自定义角色 Agent，用户可以通过 Prompt 设定角色风格。自定义 Agent 可以参与圆桌讨论，适合展示 Prompt Engineering 和 Agent 角色设定能力。

### 5. RAG 知识库流程

测试流程：

```text
进入知识库页
→ 输入内容问题
→ 点击 RAG 生成回答
→ 系统检索知识库
→ 返回知识增强回答
→ 展示命中知识片段
→ 存档当前问答
→ 删除存档
```

测试结果：通过。

系统能够根据用户问题检索 `knowledge_chunk` 表中的内容知识，并生成知识增强回答。RAG 问答结果支持本地存档和删除，便于展示知识库问答记录。

测试内容包括：

* 输入追妻火葬场题材需求

## 十、AI 应用专项测试总结

本项目不是单纯聊天页面，而是围绕 AI Agent 应用能力进行了专项测试。

测试重点包括：

1. Agent 是否能够按照角色 Prompt 发言。
2. 多个 Agent 发言是否存在明显重复。
3. 圆桌总结是否覆盖各 Agent 观点。
4. 圆桌总结是否能够带入最终内容生成。
5. Tool Calling 是否能输出结构化分析结果。
6. RAG 是否能命中知识库片段。
7. 直接生成和多 Agent 协助生成是否有明显差异。
8. 生成内容是否包含可执行的剧本结构。
9. 16 型人格是否仅作为角色风格使用，而非心理诊断结论。

测试结果表明，系统具备基础的 Multi-Agent 编排、Prompt 角色设定、RAG 知识增强、Memory 用户记忆和 Tool Calling 工具调用能力，能够支撑 AI Agent 应用开发实习岗位的项目展示。

## 十一、当前项目不足

当前项目仍存在一些可以继续优化的地方：

1. RAG 知识库规模较小，知识片段数量有限。
2. 当前 RAG 检索方式较轻量，暂未接入向量数据库。
3. 多 Agent 发言采用顺序生成方式，暂未实现复杂任务规划和自动调度。
4. AI 回复质量评估主要依赖人工检查，自动化评估能力有限。
5. 部分旧项目文件仍保留在目录中，但不影响当前核心功能运行。
6. 当前项目主要适用于本地演示，尚未进行正式部署测试。

---

## 十二、后续优化方向

后续可从以下方向继续优化：

1. 引入 Embedding 和向量数据库，提高 RAG 检索准确性。
2. 补充更多漫剧 / 短剧内容生产知识片段。
3. 增加 AI 回复质量自动评分功能。
4. 优化 Agent Prompt，减少多 Agent 发言重复。
5. 增加用户对生成结果的评分反馈。
6. 将 H2 数据库切换为 MySQL，支持正式部署。
7. 清理旧 MoodBoard 项目残留文件，使项目结构更简洁。
8. 增加更多内容生产场景，例如短视频标题生成、分镜脚本生成、评论区运营建议等。
