# AI 应用专项测试

## 一、测试目的

本文档用于记录 **AI Agent 多角色圆桌对谈与剧本生成系统** 的 AI 应用专项测试情况，重点验证系统是否具备 AI Agent 应用项目所需的核心能力，包括 Multi-Agent 多角色协作、Prompt 角色设定、Tool Calling 工具调用、RAG 知识增强、Memory 用户记忆、圆桌总结带入内容生成，以及 AI 回复质量和安全边界控制。

本专项测试不是单纯验证页面按钮是否可用，而是重点验证系统作为 AI Agent 应用项目时，是否能够体现“Agent 编排、工具调用、知识增强、上下文记忆、生成结果评估”等能力。

## 六、Multi-Agent 圆桌专项测试

### 测试目标

验证系统是否能够让多个 Agent 围绕同一议题进行协作式发言，并在最后形成总结。

### 测试步骤

1. 登录系统。
2. 进入内容页。
3. 点击“多 Agent 协助生成”。
4. 选择 INTJ、INFP、ENTP、ESTJ 四个 Agent。
5. 输入追妻火葬场题材圆桌议题。
6. 点击开始圆桌讨论。
7. 查看各 Agent 发言和圆桌总结。

### 预期结果

1. 每个 Agent 至少发言一次。
2. 不同 Agent 的表达角度存在差异。
3. 页面能够展示 Agent 名称、角色和回复内容。
4. 系统能够生成综合总结。

### 测试结果

通过。

### 测试结论

系统能够完成基础 Multi-Agent 编排流程。当前 Agent 编排方式主要为顺序发言，能够满足项目演示需求，后续可继续优化为更复杂的任务规划和动态调度。

通过。

## 十、RAG 知识库专项测试


回答应围绕前 3 秒快速建立冲突、通过误会或情感背叛制造钩子、不急于解释背景、保留后续真相反转空间等内容展开，并与知识库命中片段相关。

### 测试步骤

1. 进入知识库页面。
2. 输入测试问题。
3. 点击生成回答。
4. 查看回答内容和命中知识片段。
5. 在 H2 数据库中检查 `knowledge_chunk` 表。

### 测试结果

通过。

### 测试结论

RAG 知识库能够为内容生产问题提供知识增强支持，但当前知识库规模仍较小，后续可继续扩充。

## 十二、圆桌总结带入专项测试

### 测试目标

验证多 Agent 圆桌讨论结果是否能够真正进入最终内容生成流程，而不是仅停留在展示层。

### 测试步骤

1. 进入多 Agent 协助生成。
2. 选择 2-4 个 Agent。
3. 输入圆桌议题并生成发言。
4. 查看圆桌总结。
5. 点击“带入剧本生成”。
6. 检查内容生成页是否出现圆桌总结参考。
7. 点击生成最终方案。

### 预期结果

最终方案应包含圆桌讨论结论、不同 Agent 的观点采纳情况、最终剧本方案，以及圆桌观点如何影响剧情结构、人物设定、反转设计和分集节奏。

### 测试结果

通过。

### 测试结论

圆桌总结可以作为最终生成 Prompt 的一部分，说明系统形成了“Agent 讨论 → 总结 → 内容生产”的完整 AI 应用链路。

## 十三、直接生成与圆桌生成对比测试

### 测试目标

验证两种内容生成方式是否存在明显差异，避免用户感觉“直接生成”和“多 Agent 协助生成”结果完全一样。

| 生成方式 | 预期特点 |
|---|---|
| 直接生成方案 | 输出标准剧本方案，重点在效率和结构完整 |
| 多 Agent 协助生成 | 输出包含圆桌总结、观点采纳、不同角色建议对最终方案的影响 |

### 测试结果

通过。

### 测试结论

经过 Prompt 调整后，两种方式结果已经有明显区别。多 Agent 协助生成更适合展示 AI Agent 协作过程，直接生成更适合快速生成内容方案。

## 十五、发现的问题与优化建议

| 问题编号 | 问题描述 | 影响 | 优化建议 |
|---|---|---|---|
| AI-SUG-001 | 部分 Agent 发言存在相似表达 | 多 Agent 差异感变弱 | 优化 Prompt，明确每个 Agent 的发言角度 |
| AI-SUG-002 | RAG 知识库规模较小 | 部分问题命中片段有限 | 补充更多短剧、漫剧和运营相关知识 |
| AI-SUG-003 | 圆桌总结有时偏概括 | 总结可执行性不足 | 增加“采纳观点、风险点、执行建议”结构 |
| AI-SUG-004 | 自动化 AI 质量评测较少 | 主要依赖人工判断 | 后续可加入 AI 回复质量自动评分脚本 |
| AI-SUG-005 | Agent 编排逻辑较简单 | 当前以顺序发言为主 | 后续可增加任务拆解、投票、反驳等机制 |

## 十六、专项测试结论

本次 AI 应用专项测试验证了系统作为 AI Agent 应用项目的核心能力。

测试结果表明：

1. 系统能够完成多 Agent 圆桌讨论。
2. 不同 Agent 能够体现不同角色 Prompt。
3. 自定义 Agent 可以通过 Prompt 影响发言风格。
4. Tool Calling 可以输出结构化内容分析结果。
5. RAG 知识库可以提供知识增强回答。
6. Memory 能够记录用户上下文信息。
7. 圆桌总结可以带入最终剧本生成流程。
8. 直接生成和多 Agent 协助生成存在明显差异。
9. 系统具备一定 AI 回复质量控制和安全边界意识。

综合来看，系统已经具备 AI Agent 应用开发实习项目的基本展示能力，能够体现 Multi-Agent 编排、RAG、Memory、Tool Calling 和 Prompt Engineering 等关键能力
