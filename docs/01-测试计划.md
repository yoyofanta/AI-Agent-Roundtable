# AI Agent 多角色圆桌对谈与剧本生成系统测试计划

## 一、项目名称

AI Agent 多角色圆桌对谈与剧本生成系统

## 二、测试目的

本次测试主要针对 AI Agent 多角色圆桌对谈与剧本生成系统进行功能验证、接口验证和 AI 应用效果验证，确保系统在面试演示和本地运行环境下能够稳定完成以下核心流程：

1. 用户可以正常注册、登录、游客登录和退出登录。
2. 内容页可以提供“直接生成方案”和“多 Agent 协助生成”两种剧本生成方式。
3. 直接生成方案流程可以正常调用 Tool Calling 内容分析，并生成剧本方案。
4. 多 Agent 协助生成流程可以完成 Agent 选择、圆桌讨论、生成总结和带入剧本生成。
5. RAG 知识库可以正常检索内容知识，并生成知识增强回答。
6. RAG 问答结果可以进行本地存档和删除。
7. 自定义 Agent 可以正常创建、加入圆桌、参与讨论和删除。
8. Memory 用户记忆、RAG、Tool Calling 与多 Agent 编排流程能够配合使用。
9. 后端接口响应正常，前后端数据交互稳定。
10. 自动化测试脚本能够覆盖核心接口并生成测试报告。

## 三、测试范围

### 1. 功能测试范围

| 模块 | 测试内容 |
|---|---|
| 登录注册模块 | 注册、登录、游客登录、退出登录、登录状态保持 |
| 内容生产模块 | 选择生成方式、直接生成方案、内容分析、剧本方案生成 |
| 多 Agent 圆桌模块 | Agent 选择、性别设定、2-4 个 Agent 限制、圆桌讨论、综合总结 |
| 自定义 Agent 模块 | 创建自定义 Agent、保存、加入圆桌、删除 |
| 圆桌总结带入模块 | 圆桌总结生成后，一键带入剧本生成页面 |
| Tool Calling 模块 | 分析冲突强度、情绪基调、推荐标签和优化建议 |
| RAG 知识库模块 | 输入问题、知识检索、生成回答、展示命中片段 |
| RAG 存档模块 | 存档当前问答、展示存档、删除存档 |
| Memory 用户记忆模块 | 记录用户最近问题、历史摘要、常用 Agent 等上下文信息 |
| 页面导航模块 | 内容页、知识库页、我的页之间正常切换 |
| GitHub 项目文档 | README、测试文档、接口测试记录和测试总结内容完整 |

### 2. 接口测试范围

| 接口类型 | 测试内容 |
|---|---|
| 用户接口 | 注册、登录、游客登录、获取用户信息 |
| Agent 圆桌接口 | 提交议题和 Agent 列表，返回多 Agent 发言和总结 |
| Tool Calling 接口 | 输入剧情文本，返回冲突强度、情绪基调、标签和建议 |
| RAG 接口 | 输入知识库问题，返回知识增强回答和命中片段 |
| Memory 接口 | 获取用户记忆、清空记忆、更新记忆 |
| 自定义 Agent 接口 | 新增、查询、删除自定义角色 |
| 自动化测试接口 | 通过 Python requests 和 pytest 验证核心接口响应 |

### 3. 不在本次测试范围内的内容

以下内容暂不作为本次重点测试范围：

1. 大模型本身回复内容的绝对准确性。
2. DeepSeek API 服务端稳定性和性能。
3. 大规模并发性能测试。
4. 正式生产环境部署测试。
5. 复杂向量数据库检索效果测试。
6. 移动端多机型兼容性深度测试。
7. 旧版情绪日记、漂流瓶、盲盒等非当前项目主线功能。

### 3. DeepSeek 配置

项目支持 mock 和 real 两种模式。

开发测试阶段可使用：

```text

```

真实调用 DeepSeek 时使用：

```text
AI_MODE=real
DEEPSEEK_API_KEY=个人 API Key
AI_MODEL=deepseek-chat
```

注意：真实 API Key 不提交到 GitHub。

## 五、测试方法

### 1. 功能测试

通过浏览器手动操作页面，验证系统核心功能是否符合预期。

重点测试流程：

```text
登录 / 注册
→ 进入内容页
→ 选择直接生成方案
→ Tool Calling 内容分析
→ 生成剧本方案
```

```text
登录 / 注册
→ 进入内容页
→ 选择多 Agent 协助生成
→ 选择 2-4 个 Agent
→ 开始圆桌讨论
→ 生成圆桌总结
→ 带入剧本生成
→ 生成最终剧本方案
```

```text
进入知识库页
→ 输入问题
→ RAG 生成回答
→ 存档当前问答
→ 删除存档
```

### 2. 接口测试

使用 Postman 和 Python requests 对核心接口进行测试，验证：

1. 请求地址是否正确。
2. 请求方法是否正确。
3. 请求参数是否完整。
4. 返回状态是否正常。
5. 返回字段是否符合预期。
6. 异常参数是否有合理提示。
7. 登录 token 是否能正常传递。

### 3. 自动化测试

使用 Python requests + pytest 编写接口自动化测试脚本，覆盖核心接口流程，并生成 HTML 测试报告。

执行命令：

```bash
python -m pytest tests/test_api_pytest.py --html=tests/reports/api_report.html --self-contained-html
```

### 4. AI 应用专项测试

针对 AI Agent 应用特性进行专项验证，包括：

1. Agent 角色 Prompt 是否生效。
2. 多个 Agent 发言是否存在明显重复。
3. 圆桌总结是否覆盖各 Agent 核心观点。
4. Tool Calling 是否能识别冲突强度、情绪基调和标签。
5. RAG 回答是否与知识库内容相关。
6. 圆桌总结带入后，最终剧本方案是否明显区别于直接生成。
7. 生成结果是否包含人物设定、剧情结构、分集大纲、关键对白、标题标签和运营建议。
8. 是否避免把 16 型人格描述成心理诊断结论。

## 六、测试重点

### 1. 登录与账号隔离

重点验证：

1. 注册后是否可以直接进入系统。
2. 游客登录是否可以进入内容页。
3. 不同账号之间的本地存档是否隔离。
4. 退出登录后是否返回登录页。
5. 未登录访问系统页面是否会跳转到登录页。

### 2. 内容生成流程

重点验证：

1. 登录后是否默认进入内容选择页。
2. 内容页是否只展示“多 Agent 协助生成”和“直接生成方案”两个入口。
3. 直接生成页面是否显示完整输入表单。
4. 点击“先分析内容”后是否展示 Tool Calling 分析结果。
5. 点击“生成剧本方案”后是否生成完整内容方案。
6. Tool Calling 结果是否会带入最终生成 Prompt。

### 3. 多 Agent 协助生成流程

重点验证：

1. 是否限制选择 2-4 个 Agent。
2. 16 型人格 Agent 是否展示正常。
3. 男 / 女 / 不限性别设定是否可选。
4. 自定义 Agent 是否可以创建并加入圆桌。
5. 每个 Agent 是否至少生成一段发言。
6. 圆桌总结是否生成。
7. 点击“带入剧本生成”后是否进入直接生成页面。
8. 圆桌总结是否自动填入总结参考区域。
9. 基于圆桌总结生成的结果是否区别于直接生成结果。

### 4. RAG 知识库

重点验证：

1. 知识库页面是否可以输入任意问题。
2. RAG 是否能返回知识增强回答。
3. 是否能展示命中的知识片段。
4. 当前问答是否可以存档。
5. 存档是否可以删除。
6. H2 数据库中 `knowledge_chunk` 表是否有内容。

### 5. 接口与自动化测试

重点验证：

1. 核心接口是否能正常返回。
2. 异常参数是否有提示。
3. pytest 自动化测试是否可以全部通过。
4. HTML 测试报告是否能正常生成。

## 十二、风险与应对措施

| 风险                | 影响        | 应对措施                          |
| ----------------- | --------- | ----------------------------- |
| DeepSeek API 调用失败 | AI 回复无法生成 | 支持 mock 模式或前端兜底提示             |
| 网络不稳定             | 接口请求超时    | 增加超时时间，保留本地演示数据               |
| RAG 知识库数据不足       | 回答相关性下降   | 补充内容生产知识片段                    |
| 前端页面存在旧项目文案       | 项目定位不清晰   | 统一修改为 AI Agent 剧本生成系统         |
| 自动化测试依赖后端启动       | 测试失败      | 测试前确认后端端口和服务状态                |
| 多 Agent 回复重复      | 展示效果变弱    | 优化 Prompt，要求不同 Agent 从不同视角发言  |
| H2 数据未初始化         | RAG 无法检索  | 使用初始化类或手动检查 knowledge_chunk 表 |
